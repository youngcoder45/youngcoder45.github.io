## Contributing Guidelines

Thank you for your interest in contributing.

## How to Contribute

1. Fork the repository

2. Clone your fork
   git clone https://github.com/your-username/youngcoder45.github.io.git

3. Create a new branch
   git checkout -b feature/your-feature-name

4. Make your changes

5. Commit your changes
   git commit -m "Add: meaningful commit message"

6. Push to your fork
   git push origin feature/your-feature-name

7. Open a Pull Request

## Contribution Rules

* Keep code clean and readable
* Follow existing code style
* Avoid unnecessary dependencies
* Write meaningful commit messages
* Test your changes before submitting

## What You Can Contribute

* UI improvements
* Performance optimizations
* Bug fixes
* Accessibility improvements
* Code refactoring

## Issues

If you find a bug or want to request a feature, open an issue with:

* clear title
* detailed description
* steps to reproduce (if bug)

## Code Style

* Use TypeScript
* Use functional components
* Prefer simple and minimal solutions
* Keep components modular

## Note

This is a personal project. Major changes may not be accepted if they do not align with project goals.
